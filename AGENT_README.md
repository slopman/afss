# 🤖 Guide for AI Researchers / Agents

Welcome, Researcher. This document explains how to use the AFSS Orchestrator efficiently, specifically focusing on **saving your tokens** and getting high-quality security insights.

## 🚀 The Core Value: Token Economy

Directly reading raw JSON outputs from security tools (like Trivy or Semgrep) is **expensive and noisy**. A single scan can produce 10MB+ of JSON with 90% duplicated or irrelevant information.

**Use the Orchestrator to save 90% of your context window:**
1. **Deduplication**: We merge identical issues found by different tools or at different lines.
2. **Correlation**: We link related vulnerabilities (e.g., a vulnerable dependency linked to the code that uses it).
3. **Statistical Filtering**: We automatically lower the severity of common noise (like "Weak RNG" in test files).

## 🛠 How to Run a Research Scan

Instead of running tools manually, use the orchestrator:

```bash
# Run a full scan and get a TOKEN-OPTIMIZED report
./orchestrator scan /path/to/your/repo
```

### Where is the data?
- **Raw results**: Located in `results/raw_*.json` (Avoid reading these unless you need deep detail).
- **Optimized Report**: Located in `results/actionable_findings.json`. **Read this first!**

## 📊 Reading the Optimized Report

The `actionable_findings.json` uses a normalized format. Key fields for you:

- `confidence`: Only look at `Certain` or `Likely` if you are in a rush.
- `correlations`: This is a "map" of related issues. If you see a `same_vuln` correlation, it means we have high confidence because multiple tools flagged it.
- `fix`: We provide pre-generated fix suggestions where possible.

## 🧠 Tips for AI Researchers

1. **Don't grep raw JSON**: The orchestrator's `FindingsProcessor` has already grouped similar issues by their code hash.
2. **Focus on Correlations**: Use the `CorrelationEngine` to find patterns. A cluster of findings in one package often indicates a "security debt" area.
3. **Resource Aware**: If you run this in a CI/CD or a restricted environment, the `ResourceManager` will ensure you don't crash the host by throttling heavy tools like Semgrep.
